# 板块关系图谱 - 想法文档

## 项目愿景
构建A股板块关系图谱，分析板块轮动趋势，预测未来轮动板块。

---

## 核心设计原则
> **所有可调参数 = 默认值 + 可人工修改**
> 不同板块特性不同，需支持个性化调整

---

## 已确认需求

### 数据来源与存储
| 项目 | 确认内容 |
|------|---------|
| 数据来源 | 东财API |
| 更新时机 | 收盘后更新 |
| 数据保存 | 3个月（约60交易日） |
| 存储方式 | SQLite（约5MB） |

### 关联建立流程
```
新板块出现 → AI关系解析器分析 → 初始化默认参数 → 人工调整优化
```

### 得分算法

#### 1. 关联权重
- **范围**：-10 到 +10
- **含义**：+10强正关联，-10强负关联，0无关联
- **可调**：每对板块关系可单独调整

#### 2. 层级系数（默认值）
| 层级关系 | 默认系数 | 可调 |
|---------|---------|------|
| 同级板块 | 1.0 | ✓ |
| 相邻层级（父子） | 0.8 | ✓ |
| 跨两级 | 0.5 | ✓ |
| 包含关系 | 0.3 | ✓ |

#### 3. 时间加权（线性衰减，默认值）
| 时间范围 | 默认权重 | 可调 |
|---------|---------|------|
| 当日 | 1.0 | ✓ |
| 7日前 | 0.7 | ✓ |
| 30日前 | 0.3 | ✓ |

#### 4. 得分公式
```
板块B预测得分 = Σ (板块A涨幅% × 关联权重 × 层级系数 × 时间权重)
```

### 输出优先级
1. **可视化图谱**（节点+连线）
2. **得分排行榜**（最终输出）

> 前期为虚拟得分（未优化参数），后续根据表现迭代优化

### V0.1 - 最小原型（静态数据展示）
- [ ] 硬编码5-10个板块数据

- [ ] 显示关系类型和权重

### V0.2 - 本地数据管理
- [ ] SQLite数据库存储
- [ ] 简单增删改查界面
- [ ] 从数据库读取并渲染图谱

### V0.3 - 可视化增强
- [ ] 节点大小反映板块体量
- [ ] 连线粗细反映关联度
- [ ] 红色正关联/绿色负关联

- [ ] 接入东财API

- [ ] 自动识别新增板块

- [ ] 接入大模型API
