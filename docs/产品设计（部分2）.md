# 板块轮动预测系统 - 产品设计（部分2）

> 核心算法设计

---

## 四、核心算法

### 4.1 算法总览

```
每日盘后更新数据
    ↓
第一步：计算每个板块的"预期涨幅"（根据关联板块的实际波动）
    ↓
第二步：计算"偏差值" = 预期涨幅 - 实际涨幅（只取正值）
    ↓
第三步：计算"累计偏差得分"（近30天偏差加权累加）
    ↓
第四步：全板块按累计偏差排序 → 输出预测排行榜
```

---

### 4.2 第一步：预期涨幅计算

**公式**：
```
板块A预期涨幅 = Σ (关联板块涨幅 × 权重 × 层级系数 × 置信度)
                 ────────────────────────────────────────────
                 Σ (|权重| × 层级系数)
```

**各参数说明**：

| 参数 | 范围 | 来源 | 说明 |
|------|------|------|------|
| 关联板块涨幅 | ~-10% ~ +10% | API实时数据 | 当日收盘涨跌幅 |
| 权重 | -10 ~ +10 | AI推荐/人工设置 | 正=正相关，负=负相关 |
| 层级系数 | 0.3 ~ 1.0 | 系统默认 | 见下表 |
| 置信度 | 0 ~ 1.0 | 成交量计算 | 成交量越大越可信 |

**层级系数默认值**：

| 关系层级 | 系数 | 示例 |
|---------|------|------|
| 同一级 | 1.0 | 两个二级板块互相关联 |
| 父子关系 | 0.8 | 一级板块 ↔ 下属二级板块 |
| 跨两级 | 0.5 | 一级板块 ↔ 三级板块 |
| 包含关系 | 0.3 | 大板块包含小板块 |

**置信度计算**：
```
置信度 = min(1.0, 该板块当日净流入 / 该板块近30天日均净流入)
```
缩量波动的影响力被降低，放量波动的影响力保持。

**计算示例**：

> 计算"锂电池"的预期涨幅
>
> | 关联板块 | 今日涨幅 | 权重 | 层级系数 | 置信度 | 贡献 |
> |---------|---------|------|---------|--------|------|
> | 锂矿 | +3% | 8 | 1.0 | 0.9 | +21.6 |
> | 新能源汽车 | +2% | 6 | 0.8 | 1.0 | +9.6 |
> | 光伏 | -1% | 3 | 0.5 | 0.6 | -0.9 |
> | 权重总和 | | | | | 12.8 |
>
> 预期涨幅 = (21.6 + 9.6 - 0.9) / 12.8 = **+2.37%**

---

### 4.3 第二步：偏差计算

```
偏差值 = max(0, 预期涨幅 - 实际涨幅)
```

**关键设计：只取正偏差**

| 情况 | 含义 | 处理 |
|------|------|------|
| 预期+3%，实际+1% | 偏差+2%，还没涨到位 | ✅ 记录偏差+2%（补涨机会） |
| 预期+2%，实际+4% | 涨超预期 | ⭕ 偏差=0（中性，不扣分） |
| 预期-1%，实际-1% | 正常下跌 | ⭕ 偏差=0（中性） |

> 设计理由：负偏差（涨超预期）可能是板块自身有利好，不代表利空，所以不扣分。

---

### 4.4 第三步：累计偏差得分

**公式**：
```
累计偏差得分 = Σ (每日偏差 × 时间权重)
```

**时间衰减**（线性衰减，30天窗口）：
```
时间权重 = 1.0 - (距今天数 / 30) × 0.9

今天: 1.0
3天前: 0.91
7天前: 0.79
14天前: 0.58
30天前: 0.10
30天之前: 不计入
```

**示例：追踪"锂电池"一周的累计偏差**

| 日期 | 预期 | 实际 | 偏差 | 时间权重 | 加权偏差 | 累计 |
|------|------|------|------|---------|---------|------|
| Day1(7天前) | +2.3% | +0.5% | 1.8% | 0.79 | 1.42 | 1.42 |
| Day2(6天前) | +1.5% | +0.3% | 1.2% | 0.82 | 0.98 | 2.40 |
| Day3(5天前) | +0.8% | +0.2% | 0.6% | 0.85 | 0.51 | 2.91 |
| Day4(4天前) | +1.0% | +3.5% | 0 | 0.88 | 0 | 2.91 |
| Day5(3天前) | +0.5% | +0.1% | 0.4% | 0.91 | 0.36 | 3.27 |
| Day6(2天前) | +1.2% | +0.8% | 0.4% | 0.94 | 0.38 | 3.65 |
| Day7(今天) | +2.0% | +0.5% | 1.5% | 1.00 | 1.50 | **5.15** |

> Day4锂电池补涨了一波（实际+3.5% > 预期+1.0%），偏差为0。
> 但其他天持续"欠账"，累计偏差仍在上升 → 预示还有补涨空间。

---

### 4.5 第四步：排行输出

```
所有板块按累计偏差得分从高到低排序
前10名 = "最可能轮动补涨的板块"
```

---

### 4.6 回测验证逻辑

```
取过去60天历史数据
对每一天T（从第31天开始）：
  1. 用T-1日的数据计算预测排行
  2. 取预测排行前10名
  3. 看T日这10个板块的实际涨幅
  4. 如果实际涨幅排名也在前20名 → 命中

准确率 = 总命中数 / (测试天数 × 10) × 100%
```

---

### 4.7 权重管理四层体系

```
┌─────────────┐
│ 1. 系统默认   │  新关联创建时的初始权重（默认5）
├─────────────┤
│ 2. AI推荐    │  AI分析后推荐的权重
├─────────────┤
│ 3. 人工调整   │  用户修改的权重（可锁定🔒）
├─────────────┤
│ 4. AI回测优化  │  AI根据回测结果优化未锁定的权重
└─────────────┘

优先级：人工锁定 > AI回测优化 > AI推荐 > 系统默认
```

**AI与锁定的交互规则**：
- AI分析和优化时**读取所有参数**（包括锁定的）用于综合分析
- AI**仅修改未锁定的参数**
- 对锁定参数，AI**给出建议但不执行**，由用户决定是否采纳

---

### 4.8 算法参数一览

| 参数名 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| time_decay_days | 30 | 7~90 | 时间衰减窗口（天） |
| time_decay_min | 0.1 | 0~0.5 | 最远日期的最低权重 |
| deviation_mode | positive_only | - | 偏差模式：只取正偏差 |
| backtest_period | 60 | 30~180 | 回测周期（天） |
| ranking_top_n | 10 | 5~30 | 预测排行前N名 |
| backtest_hit_range | 20 | 10~50 | 回测命中范围（实际前N名算命中） |

> 所有参数可在设置面板中调整
