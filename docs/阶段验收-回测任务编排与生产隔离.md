## 1. 阶段目标
- 跑通回测任务全链路：`触发 -> 排队 -> 执行 -> 结果查询`
- 验证任务状态机一致性：`queued/running/completed/failed/cancelled`
- 验证 `run_type` 隔离：回测链路不污染生产排行

## 2. 进入下一阶段的硬门槛
1. `POST /api/backtest/run` 返回 `status=processing` 且包含 `run_id`
2. `GET /api/backtest/jobs/{run_id}` 可观察到任务终态（`completed/failed/cancelled`）
3. `POST /api/backtest/jobs/{run_id}/cancel` 仅在允许状态生效，返回可解释信息
4. `POST /api/backtest/jobs/{run_id}/retry` 仅在 `failed/cancelled` 生效，返回 `new_run_id`
5. `GET /api/backtest/results?run_id=...` 与 `GET /api/backtest/results/{target_date}?run_id=...` 可稳定返回
   且返回 `run_id/params_snapshot`，支持按任务复现
6. `GET /api/ranking?run_type=prod` 与 `run_type=backtest` 查询结果可区分
7. 数据质量门控回归基线持续通过（至少 1 轮）
8. 趋势与单日明细口径一致（`hits/alpha` 抽样一致）

## 3. 最小验收步骤（PowerShell）

### 3.1 触发任务并轮询状态
```powershell
$base = "http://127.0.0.1:8000"
$run = Invoke-RestMethod -Method Post "$base/api/backtest/run?days=30"
$runId = $run.run_id
"run_id=$runId"

for ($i = 0; $i -lt 60; $i++) {
  $job = Invoke-RestMethod "$base/api/backtest/jobs/$runId"
  "status=$($job.status)"
  if ($job.status -in @("completed","failed","cancelled")) { break }
  Start-Sleep -Seconds 2
}
```

### 3.2 取消与重试
```powershell
$cancel = Invoke-RestMethod -Method Post "$base/api/backtest/jobs/$runId/cancel"
$detail = Invoke-RestMethod "$base/api/backtest/jobs/$runId"
$retry = Invoke-RestMethod -Method Post "$base/api/backtest/jobs/$runId/retry"
$newRunId = $retry.new_run_id
```

### 3.3 结果与隔离检查
```powershell
Invoke-RestMethod "$base/api/backtest/results?limit=10&run_id=$runId"
```

## 6. Web 可见验收点（回测任务状态机）
以下项目可直接在页面验证，不依赖代码阅读：

1. 进入 `回测复盘` 页，出现 `回测任务状态机` 面板。
   面板包含：`启动回测任务`、`刷新状态`、任务列表表格。

2. 点击 `启动回测任务` 后，出现 toast，文案包含：
   `任务ID(run_id)` 等可追踪信息。

3. 任务列表中，状态标签可见且属于：
   `排队中/运行中/已完成/已失败/已取消`。

4. 对 `已取消/已失败` 任务点击 `重试` 后：
   顶部出现新的 `run_id`，且 `当前任务` 字段更新为新任务。

5. 点击任一行 `详情` 按钮后：
   页面 toast 显示“任务 xxx 当前状态：yyy”。

6. 首页 `链路进度（阶段锁）` 的第 5 步文案应同步展示：
   `回测任务：<run_id> · <状态>`，用于非代码视角的阶段确认。

7. 在任务列表点击 `查看结果` 后：
   `结果视角：<run_id>` 文案会切换，`走势视图/单日明细` 按该 `run_id` 查询。

8. 点击 `🧪 校验口径` 后：
   页面出现 `口径校验：通过/失败` 状态文案，且可在悬浮提示查看 `run_id/抽样数/不一致数`。
