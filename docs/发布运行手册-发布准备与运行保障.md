# 发布运行手册（发布准备与运行保障）
更新时间：2026-02-27 15:45

## 1. 适用范围
- 适用于阶段：`发布准备与运行保障`
- 目标：在不改动核心策略逻辑前提下，完成发布前检查、异常回滚与证据留存。

## 2. 发布前执行顺序（严格顺序）
1. 刷新数据并确认质量门控可发布  
命令：`POST /api/sectors/refresh`  
通过标准：`publish_allowed=true`

2. 执行计算评分并确认排行榜可见  
操作：首页点击 `执行计算评分`  
通过标准：排行榜有有效结果（非空）

3. 回测复盘确认最近任务 completed  
操作：进入“回测复盘”查看最新 `run_id`  
通过标准：最新任务状态为 `completed`

4. 配置版本确认 refeeded 可追溯  
操作：进入“系统设置 -> 配置版本化”  
通过标准：最新已应用版本状态为 `refeeded`

## 3. 一键验收命令
```powershell
powershell -ExecutionPolicy Bypass -File scripts/run_release_stage_acceptance.ps1 -BaseUrl http://127.0.0.1:8000
```
- 通过标准：报告中 `passed=true`
- 报告位置：`reports/release_stage_acceptance_*.json`

## 4. 异常回滚口径
1. 禁止继续发布  
- 若 4 项检查任一未通过，立即停止发布动作。

2. 版本回滚  
- 在“系统设置”切回上一个稳定配置版本（保留回滚前后版本号）。

3. 回测定位  
- 在“回测复盘”记录异常对应 `run_id`、日期窗口与关键漂移指标。

4. 证据留存  
- 保存回滚前后截图、验收报告和异常说明，归档到 `reports/` 与阶段文档。

## 5. Web 端对应入口
- 首页面板：`发布准备检查`
- 首页面板：`发布运行手册与回滚口径`
- 页面入口：`回测复盘`、`系统设置`
