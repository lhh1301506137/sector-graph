# 阶段验收：API 拉取 -> 数据质量门控
更新时间：2026-02-26

阶段状态：`已完成（可切阶段）`
最新通过报告：`reports/quality_gate_acceptance_20260226_192648.json`

## 1. 阶段目标
- 验证主链路：`刷新数据 -> 质量计算 -> 发布门控`
- 验证阻断链路：当质量不达标时，`publish=true` 必须被阻断且返回可解释原因
- 验证失败明细链路：`/api/data-quality/failed` 可返回失败样本与原因聚合

## 2. 进入下一阶段的硬门槛
1. `GET /api/data-quality/latest` 返回 `failed_rows/publish_allowed/publish_blocked_reasons`
2. `POST /api/scoring/run` 在 `publish_allowed=true` 时返回 `200`
3. `POST /api/scoring/run` 在 `publish_allowed=false` 时返回 `400`
4. `GET /api/data-quality/failed` 在失败场景可返回非空样本
5. `GET /api/data-quality/trend` 返回结构包含 `failure_buckets` 核心字段

## 3. 一键验收命令
在项目根目录执行：

```powershell
.\scripts\run_quality_gate_acceptance.ps1
```

或执行批处理入口：

```bat
check_quality_gate_acceptance.bat
```

默认流程：
1. 启动临时服务（`127.0.0.1:18001`）
2. 执行 `refresh -> data-quality/latest -> scoring publish gate`
3. 执行阻断场景回归（提高 `quality_min_total_rows` 后恢复）
4. 执行失败行场景回归（收紧 `quality_turnover_abs_max` 后恢复）
5. 输出验收报告到 `reports/`

## 7. API 字段白名单（P0，跑通全链路必需）
字段口径采用“标准层”统一命名，至少包含以下字段：

| 字段 | 含义 | 是否必需 |
|---|---|---|
| `trade_date` | 交易日期 | 是 |
| `source_name` | 数据来源（sina/akshare/eastmoney/tushare） | 是 |
| `name` | 板块名称 | 是 |
| `category_type` | 板块类型（行业/概念） | 是 |
| `daily_change` | 板块当日涨跌幅 | 是 |
| `turnover` | 板块换手率 | 是 |
| `net_amount` | 板块净流入/净额 | 是 |
| `lead_stock` | 领涨股名称 | 否（建议） |
| `lead_stock_change` | 领涨股涨跌幅 | 是 |
| `api_id` | 外部源主键/代码 | 否（建议） |

当前默认 `quality_required_fields`：
- `name,category_type,daily_change,net_amount,turnover,lead_stock_change`

## 8. 单位口径（标准层）
- `daily_change`：单位 `%`
- `lead_stock_change`：单位 `%`
- `turnover`：单位 `%`
- `net_amount`：单位 `亿元`

数据源归一化规则（当前实现）：
- `sina`：`avg_changeratio`、`ts_changeratio` 从小数换算到 `%`；`netamount` 归一到 `亿元`；`turnover` 归一到 `%`
- `eastmoney`：`f3`、`f8`、`f136` 直接按 `%` 处理；`f62` 归一到 `亿元`
- `akshare`：字段按页面语义解析并统一到标准层口径
- `tushare`：`pct_change/pct_change_stock` 作为 `%`；`net_amount`（或净买卖差）归一到 `亿元`

## 9. 推荐默认门控阈值（面向“可发布”生产基线）
建议作为新环境默认值：

| 配置键 | 推荐值 | 说明 |
|---|---:|---|
| `quality_max_failed_rows` | `25` | 允许少量异常，不因单源短抖动全量阻断 |
| `quality_min_total_rows` | `180` | 防止拉取量过低仍误发布 |
| `quality_stale_minutes` | `240` | 4小时新鲜度窗口 |
| `quality_require_freshness_for_publish` | `1` | 发布前强制新鲜度通过 |
| `quality_daily_change_abs_max` | `20` | 板块日涨跌幅绝对值上限 |
| `quality_lead_stock_change_abs_max` | `25` | 领涨股涨跌幅绝对值上限 |
| `quality_turnover_abs_max` | `30` | 板块换手率绝对值上限 |
| `quality_net_amount_abs_max` | `500` | 板块净额绝对值上限（亿元） |

说明：
- 若使用“极严模式”（如 `quality_turnover_abs_max=1` 且 `quality_max_failed_rows=0`），通常会触发大量阻断，属于测试/排障模式，不建议作为生产默认值
