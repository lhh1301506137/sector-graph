## 六、AI交互设计

### 6.1 AI在系统中的角色

AI是**辅助工具**，不是决策者。核心定位：
- 初始化关联关系（减少人工工作量）
- 优化建议（基于回测数据）
- 最终参数由用户掌控

### 6.2 场景一：新板块关联分析

**流程**：
```
用户点击"🤖 AI分析"
  ↓
弹窗显示："检测到15个新板块，预计调用AI约80次，约消耗0.3元，耗时约2分钟。继续？"
  ↓
第一轮：AI将新板块分入产业群（1次调用）
  ↓
第二轮：只分析同产业群内的板块对（大幅减少组合数）
  ↓
结果进入"待确认"队列
  ↓
用户浏览：✅ 接受 / ❌ 拒绝 / ✏️ 修改 / 📋 全部接受
  ↓
确认的关联写入数据库
```

**Prompt模板**：
```
你是A股板块关系分析专家。请分析以下板块对的关联关系。

板块A: {板块名A}
板块B: {板块名B}

可选关系类型：供应、应用、配套、技术同源、成本影响、
需求影响、替代、互补、竞争、政策联动、无关

请返回JSON，只返回JSON不要其他内容：
{
  "hasRelation": true/false,
  "type": "关系类型",
  "weight": 权重(-10到10的整数),
  "direction": "A→B" 或 "B→A" 或 "A↔B",
  "reason": "一句话分析理由"
}
```

### 6.3 场景二：回测后优化权重

**流程**：
```
用户运行回测 → 输出准确率（如42%）
  ↓
点击"AI优化建议"
  ↓
系统筛选偏差最大的Top20预测 + 相关权重（含锁定状态）
  ↓
发送给AI分析
  ↓
AI返回调整建议（包括对锁定参数的建议）
  ↓
系统展示：
  - 未锁定参数：[一键应用] / [忽略]
  - 锁定参数：显示建议 + "如需修改请手动解锁"
```

**Prompt模板**：
```
以下是板块轮动预测的回测结果，当前准确率{accuracy}%。
请分析并给出关联权重的优化建议。

偏差最大的预测（预测高分但未上涨的板块）：
{top_deviation_list}

相关的关联权重：
{related_weights_with_lock_status}

请返回JSON数组，只返回JSON不要其他内容：
[
  {
    "source": "源板块名",
    "target": "目标板块名",
    "current_weight": 当前权重,
    "suggested_weight": 建议权重,
    "is_locked": true/false,
    "reason": "调整理由"
  }
]
```

### 6.4 AI安全机制

| 机制 | 说明 |
|------|------|
| 成本预估 | 操作前显示预计调用次数和费用 |
| 待确认队列 | AI结果不直接入库，需用户确认 |
| 锁定保护 | AI不修改锁定参数，仅给建议 |
| JSON容错 | 多层解析：直接解析→去markdown标记→正则提取→跳过 |
| 调用日志 | 记录每次输入输出，可回溯 |
| 分批处理 | 大量分析时分批调用，支持中断 |

```

│

│   ├── index.html              # 单页应用
│   ├── style.css               # 样式
```

### 7.2 前后端通信

```
前端 (localhost:8000)
  │
  ├── GET  /api/sectors          → 获取板块列表
  ├── POST /api/sectors/refresh  → 从API刷新数据
  │
  ├── GET  /api/relations        → 获取关联列表
  ├── POST /api/relations        → 添加关联
  ├── PUT  /api/relations/{id}   → 修改关联（含锁定）
  │
  ├── GET  /api/ranking          → 获取预测排行
  ├── POST /api/scoring/run      → 运行得分计算
  │
  ├── POST /api/ai/analyze       → AI分析新板块
  ├── GET  /api/ai/pending       → 获取待确认列表
  ├── POST /api/ai/confirm       → 确认AI结果
  │
  ├── POST /api/backtest/run     → 运行回测
  ├── POST /api/ai/optimize      → AI优化建议
  │
  ├── GET  /api/config           → 获取配置
  └── PUT  /api/config           → 更新配置
```

### 7.3 启动脚本

**setup.bat**（首次运行）：
```bat
@echo off
echo 正在创建Python虚拟环境...
python -m venv venv
call venv\Scripts\activate
echo 正在安装依赖...
pip install -r server\requirements.txt
echo 安装完成！请运行 start.bat 开始使用。
pause
```

**start.bat**（每次启动）：
```bat
@echo off
call venv\Scripts\activate
echo 正在启动板块轮动预测系统...
start http://localhost:8000
python server\app.py
```
