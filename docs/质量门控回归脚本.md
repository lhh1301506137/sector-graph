# 质量门控回归脚本

当前用于回归基线（覆盖 `API 拉取 -> 数据质量门控`）：

- 根目录一键入口：`check_quality_gate.bat`
- 可参数入口：`scripts/run_quality_gate_smoke.ps1`
- 核心脚本：`scripts/quality_gate_smoke.py`
- 阶段验收入口（输出报告）：`scripts/run_quality_gate_acceptance.ps1`

## 1. 一键执行

在项目根目录运行：

```bat
check_quality_gate.bat
```

默认行为：

1. 启动临时服务（`127.0.0.1:18001`）
2. 执行 `refresh -> data-quality -> scoring(publish=true)` 链路
3. 自动断言门控结果：
   - `publish_allowed=true` 时，`/api/scoring/run` 必须返回 `200`
   - `publish_allowed=false` 时，`/api/scoring/run` 必须返回 `400`
4. 自动执行“阻断场景”：
   - 临时把 `quality_min_total_rows` 提高到 `total_rows+1`
   - 断言 `publish_allowed=false` 且 `/api/scoring/run` 返回 `400`
   - 自动恢复原配置并校验恢复后门控状态一致
5. 自动执行“失败行场景”：
   - 临时把 `quality_turnover_abs_max` 降到较低值（默认 `1.0`），制造失败行
   - 断言 `failed_rows>0`，且 `publish_blocked_reasons` 包含 `failed_rows_threshold`
   - 调用 `/api/data-quality/failed` 断言有失败明细
   - 断言 `/api/scoring/run` 返回 `400`
   - 自动恢复原配置并刷新恢复数据
6. 自动退出并回收临时服务

## 2. PowerShell 参数模式

```powershell
.\scripts\run_quality_gate_smoke.ps1 -StartServer -Port 18001 -Source sina -ExerciseBlockedCase -ExerciseFailedRowsCase
```

常用参数：

- `-StartServer`：本次回归自动启动临时服务
- `-BaseUrl`：复用已有服务地址（不加 `-StartServer` 时生效）
- `-Source`：刷新源，默认 `sina`
- `-SkipRefresh`：跳过刷新，仅验证现有数据门控
- `-ExerciseBlockedCase`：执行“强制阻断并恢复”回归
- `-ExerciseFailedRowsCase`：执行“失败行阈值拦截 + 明细接口”回归
- `-ExpectPublishAllowed auto|true|false`：额外校验门控预期

## 3. 输出判定

- 成功：输出 `[PASS] quality gate smoke test`
- 失败：输出 `[FAIL] ...` 并返回非零退出码

## 4. 失败明细排查接口

用于查看质量失败样本（仍属于 `API拉取 -> 数据质量门控` 阶段）：

```text
GET /api/data-quality/failed
```

常用参数：

- `target_date=YYYY-MM-DD`
- `reason=anomaly_turnover_abs_gt_1.0`（按原因精确筛选）
- `category_type=行业|概念`
- `search=关键词`
- `limit=1..500`

返回包含：

- `total_failed_rows`
- `reason_distribution`
- `items[]`（含 `sector_name`, `quality_reason`, 当日行情字段）

## 5. 质量趋势接口

用于查看最近 N 天门控结果趋势：

```text
GET /api/data-quality/trend?days=10
```

返回包含：

- `items[].date`
- `items[].total_rows / failed_rows / failed_ratio`
- `items[].publish_allowed`
- `items[].blocked_rules`
- `items[].top_reason`
