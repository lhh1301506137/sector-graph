## 1. 归档触发条件
- 首页阶段卡第 10 步显示：`发布后运行观察与收官验收`。
- “发布后运行观察与收官验收”面板达到：`收官观察通过 5/5`。
- 阶段提示文案显示：`可进入收官归档`。

## 2. 最小归档清单
1. 发布基线验收报告  
`reports/release_stage_acceptance_*.json`
2. 发布后观察验收报告  
`reports/post_release_observe_acceptance_*.json`
3. Web 可见证据截图  
`reports/post_release_stage_step1_desktop_*.png`
4. 当前阶段说明文档  
`docs/阶段验收-发布后运行观察与收官验收.md`

## 3. 归档执行建议
1. 先运行命令：
```powershell
powershell -ExecutionPolicy Bypass -File scripts/run_release_stage_acceptance.ps1 -BaseUrl http://127.0.0.1:8000
powershell -ExecutionPolicy Bypass -File scripts/run_post_release_observe_acceptance.ps1 -BaseUrl http://127.0.0.1:8000
```
2. 确认两份报告 `passed=true`。
3. 保存当次首页截图到 `reports/`，并在阶段验收文档里登记报告文件名。

## 4. 未通过处置口径
- 若发布基线验收未通过：先回到“发布运行手册与回滚口径”面板补齐 4 步。
- 若发布后观察验收未通过：先处置“发布后运行观察”面板中的“待处置”项。
- 处置完成后再重复第 3 节命令，不通过不归档。
