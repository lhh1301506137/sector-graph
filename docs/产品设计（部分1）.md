# 板块轮动预测系统 - 产品设计文档

> 版本：1.0 | 日期：2026-02-12 | 状态：讨论共识版

---

## 一、产品概述

### 1.1 核心目标
构建一个**A股板块轮动预测系统**，通过分析板块间关联关系和实时涨跌数据，利用"偏差累计"算法预测下一个可能轮动上涨的板块。

### 1.2 核心逻辑
> 如果关联板块已经涨了，但目标板块还没跟上，说明它有"欠账"，后面可能补涨。

### 1.3 使用场景
- 用户：个人使用，A股投资研究
- 频率：每日盘后打开，手动触发操作
- 部署：本地运行（Windows）

### 1.4 工作流
```
每日盘后 → 打开应用 → 点击"刷新数据" → 自动更新全板块涨跌
→ 系统计算偏差得分 → 输出预测排行榜
→ 可选：AI分析新板块 / 运行回测 / 手动调参
```

## 三、数据模型

### 3.1 sectors（板块表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT UNIQUE | 板块名称 |
| category_type | TEXT | "行业" / "概念" |
| api_id | TEXT | API标识（如 gn_ldc） |
| level | INTEGER | 层级 1/2/3 |
| parent_id | INTEGER FK | 父板块ID |
| is_active | BOOLEAN | 是否在图谱中显示（默认true） |
| is_favorited | BOOLEAN | 是否关注（默认false） |
| created_at | DATETIME | 创建时间 |

### 3.2 relations（关联关系表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| source_id | INTEGER FK | 源板块 |
| target_id | INTEGER FK | 目标板块 |
| type | TEXT | 10种关系之一 |
| weight | FLOAT | 权重 -10 ~ +10 |
| direction | TEXT | "A→B" / "A↔B" |
| level_coefficient | FLOAT | 层级系数（默认1.0） |
| is_locked | BOOLEAN | 用户锁定标记 |
| source | TEXT | 来源："system" / "ai" / "manual" |
| valid_from | DATE | 生效起始（预留） |
| valid_to | DATE | 生效截止（预留） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**关系类型（10种）**：供应、应用、配套、技术同源、成本影响、需求影响、替代、互补、竞争、政策联动

### 3.3 daily_data（每日数据表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| sector_id | INTEGER FK | 板块ID |
| date | DATE | 日期（联合唯一：sector_id + date） |
| daily_change | FLOAT | 实际涨跌幅 % |
| expected_change | FLOAT | 预期涨幅 %（算法计算） |
| deviation | FLOAT | 偏差值（预期-实际，取正） |
| cumulative_deviation | FLOAT | 累计偏差得分 |
| net_amount | FLOAT | 净流入（亿） |
| turnover | FLOAT | 换手率 |
| lead_stock | TEXT | 领涨股名称 |
| lead_stock_change | FLOAT | 领涨股涨幅 % |

> 每天约300条，90天 ≈ 27000条，SQLite完全胜任

### 3.4 predictions（预测记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| sector_id | INTEGER FK | 板块ID |
| date | DATE | 预测日期 |
| score | FLOAT | 累计偏差得分 |
| rank | INTEGER | 当日排名 |

### 3.5 backtest_results（回测结果表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| run_date | DATETIME | 运行时间 |
| period_start | DATE | 回测起始 |
| period_end | DATE | 回测结束 |
| accuracy | FLOAT | 准确率 % |
| details | TEXT (JSON) | 每日详细结果 |

### 3.6 config（配置表）

| 字段 | 类型 | 说明 |
|------|------|------|
| category | TEXT | 分组："ai" / "algo" / "user" |
| key | TEXT | 配置键（联合主键：category + key） |
| value | TEXT (JSON) | 配置值 |

配置示例：
- `ai.provider` = "deepseek"
- `ai.api_key` = "sk-xxx"
- `algo.time_decay_days` = 30
- `algo.deviation_mode` = "positive_only"
