# 数据状态机（讨论稿）

> 目的：把“板块轮动系统”里每类数据在生命周期中的变化说清楚，避免功能互相覆盖、数据口径不一致。

## 1. 板块（Sector）状态机

### 状态定义
- `new_detected`：API 新发现，尚未完成关联分析
- `ai_pending`：已进入 AI 关联建议队列，待用户确认
- `active`：已确认可参与评分与排行
- `disabled`：人工停用，不参与评分
- `archived`：长期废弃，仅保留历史

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| 无 | API 刷新发现新板块 | `new_detected` | 以 `api_id + category_type` 去重 |
| `new_detected` | 触发 AI 分析 | `ai_pending` | 生成待确认关联建议 |
| `ai_pending` | 用户确认建议（可编辑） | `active` | 进入生产评分链路 |
| `active` | 用户停用 | `disabled` | 保留历史，不参与评分 |
| `disabled` | 用户恢复 | `active` | 重新参与评分 |
| `active/disabled` | 用户归档 | `archived` | 默认只读 |

---

## 2. 关联（Relation）状态机

### 状态定义
- `pending`：AI 建议尚未确认（在 `pending_ai_suggestions`）
- `confirmed`：用户确认并写入 `relations`
- `locked`：用户锁定，不允许 AI 优化覆盖
- `inactive`：逻辑失效，暂不参与评分

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| 无 | AI 产出建议 | `pending` | 不直接进生产 |
| `pending` | 用户确认 | `confirmed` | 去重后写入 |
| `confirmed` | 用户锁定 | `locked` | 高优先级，AI 只读 |
| `locked` | 用户解锁 | `confirmed` | 恢复可调 |
| `confirmed/locked` | 用户停用或系统清理 | `inactive` | 不参与评分 |

---

## 3. 日线数据（DailyData）状态机

### 状态定义
- `raw_ingested`：行情已入库（涨跌、净流入、换手等）
- `quality_failed`：质量校验失败（缺字段/异常值）
- `ready_for_scoring`：质量通过，可评分
- `scored`：已写入 `expected_change/deviation/cumulative_deviation`

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| 无 | API 入库 | `raw_ingested` | 同日同板块唯一 |
| `raw_ingested` | 质量门控失败 | `quality_failed` | 不进入评分 |
| `raw_ingested` | 质量门控通过 | `ready_for_scoring` | 等待评分任务 |
| `ready_for_scoring` | 运行评分 | `scored` | 写回三项计算字段 |

---

## 4. 评分结果（Prediction）状态机

### 状态定义
- `draft`：某次计算产生的临时结果（可用于回测）
- `published`：正式发布到生产排行
- `invalidated`：被新版本参数或新数据替代

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| 无 | 运行评分（非发布模式） | `draft` | 回测或试算用 |
| `draft` | 发布排行 | `published` | 对前端可见 |
| `published` | 新一轮发布 | `invalidated` | 旧记录保留供追溯 |

---

## 5. 回测任务（Backtest）状态机

### 状态定义
- `queued`：已提交任务
- `running`：执行中
- `completed`：完成
- `failed`：失败
- `cancelled`：取消

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| 无 | 用户发起回测 | `queued` | 记录参数快照 |
| `queued` | worker 开始执行 | `running` | 后台任务 |
| `running` | 正常完成 | `completed` | 生成结果明细 |
| `running` | 异常中断 | `failed` | 记录错误原因 |
| `queued/running` | 用户取消 | `cancelled` | 可重试 |

---

## 6. 配置（Config）状态机

### 状态定义
- `draft`：前端编辑中（未保存）
- `saved`：已保存，待下一轮任务生效
- `applied`：已被某次评分/回测实际使用
- `rolled_back`：已回滚到上一版本

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| 无 | 用户修改表单 | `draft` | 本地状态 |
| `draft` | 点击保存 | `saved` | 写库并版本化 |
| `saved` | 运行评分/回测 | `applied` | 绑定 `run_id` |
| `applied` | 用户回滚 | `rolled_back` | 可审计 |

## 8. 数据源连通与降级状态机（2026-02-25 新增）

> 目的：让“数据源是否可用、是否降级”可观察、可解释，避免刷新成功但口径不清。

### 状态定义
- `unchecked`：未检测（进入设置页默认状态）
- `ok`：主路径可用
- `degraded`：主路径不可用，已启用兜底路径
- `error`：主路径与兜底路径均失败

### 状态转移
| 当前状态 | 触发事件 | 下一个状态 | 备注 |
|---|---|---|---|
| `unchecked` | 用户点击连通性检查 | `ok/degraded/error` | 按实时网络结果判定 |
| `ok` | 主路径失败但兜底成功 | `degraded` | 不中断主链路 |
| `degraded` | 主路径恢复可用 | `ok` | 自动恢复 |
| `ok/degraded` | 主路径与兜底均失败 | `error` | 需要人工处理 |

### 同步元数据建议
- 在同步状态中保留：
  - `requested_source_name`
  - `source_name`
  - `verify_source_name`
  - `source_degraded`
- 前端展示要求：
  - 未检查时显示 `未检测`
  - 降级时明确显示 `OK(降级)` 与降级原因
